How Abribook Works

Welcome to SYG Abribook

Abribook is the intake logging and code lookup tool for Sell Your Gadgets. It helps you quickly identify storage drives and RAM modules, log their condition, and build a manifest that Claude uses to generate your Shopify listings.

This tutorial walks you through the full process — from picking up the item to handing off to Claude. It only takes a minute.

1Select your name & batch

At the top of the screen, select your name from the dropdown and enter your batch number (e.g. RTL0623). These lock for the rest of the session.

Your initial will appear in every SKU you generate — for example, Juan's entries will read NJ02-J-RTL0623-1 — so listings are always traceable back to you.

2Inspect the item

For SSDs: Plug the drive into the test rig and open CrystalDiskInfo. Note the health %, serial number, and any physical damage you can see on the drive.

For RAM: Seat the module in its corresponding slot on the RAM tester and observe the indicator lights. Each light represents a module within the RAM — a fully bright light means that section is healthy. A dim light indicates that section may not be functioning fully, which can cause slowdowns or failures under real workloads even if the stick appears to work. Note the serial number and any visible damage.

3Look up the code

Use the search bar on the left to find the item's Abribook code — search by drive name (e.g. 970 EVO) or by code (e.g. S2990P). Switch between the Storage and RAM tabs as needed.

If the item isn't in the list yet, use Add a new code to register it. Follow the code recipe shown — keep it consistent with existing codes.

4Fill in the draft entry

In the Draft Log Entries panel on the right, enter the code, health % or Pass/Fail result, serial number, and any notes. The preview line updates live so you can see exactly what will be copied.

Add one entry per unit. For RAM pairs, enable Counting pairs and enter the quantity.

5Copy all & hand off to Claude

Once you've logged all items for the batch, hit Copy All and open Claude. Paste the entries, then run the SYG Listing Generation skill to have Claude generate the listings from the manifest.

⚠️ Pricing warning: Pricing for RAM must always be double or even triple checked — especially for server-grade and enterprise modules. Market value can vary significantly by spec and rank. When in doubt, research sold comps before publishing.

Universal Page Tutorial

What Universal does

The Universal page is for items that are not RAM or storage: CPUs, GPUs, consoles, laptops, desktops, phones, and motherboards.

You are not writing the final listing here. You are building a clean manifest entry that Claude can later use to fix the website-generated title, HTML description, and metafields in bulk.

1Start with the Website SKU

The company website creates the rough listing first. Copy that listing's SKU into Abribook.

Website SKU is the matching key. If it is wrong or missing, Claude cannot safely match your item to the correct website listing.

2Choose what is in your hand

Use What's In My Hand Right Now? to choose the item type. Abribook changes the required fields, checklist, confidence score, and HTML preview based on that choice.

  • Laptop/Desktop: CPU, RAM, storage, GPU, OS, testing, condition.
  • Phone: storage, carrier/lock status, battery, screen, testing.
  • GPU/Motherboard: model, testing, damage, ports/socket details.

3Describe it like a human

Type the real specs and condition in plain English. Abribook is designed so a lister can quickly write what they see without thinking about spreadsheet columns.

Good example: Dell Precision 5570, i7-12800H, 32GB RAM, 1TB SSD, RTX A2000. Boots to BIOS. No charger. Minor lid scratches.

4Use confidence + Needs Research

The Listing Confidence meter tells you if the entry has enough information for Claude to prepare listing data. High confidence means it is probably ready for review. Low confidence means important facts are missing.

Use Needs Research when the item is unclear, locked, damaged in a confusing way, missing model/specs, or needs pricing verification. It keeps bad guesses out of the final listing workflow.

5Photos come after data

After the required data is filled and the entry reaches ready/high confidence, Abribook will tell you it is time to take photos.

That keeps the flow clean: identify → test → enter data → take photos → add to manifest queue → export the whole batch to Claude.

6Review the whole batch at the end

Each item goes into the manifest queue first. At the end, review everything together, sync it to the Universal Sheet, then copy the Claude manifest-prep prompt.

Claude should use the Website SKU to return corrected titles, HTML descriptions, metafields, and missing-info warnings for bulk listing edits.
